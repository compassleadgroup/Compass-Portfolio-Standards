Generate the closing assets to pitch a contractor on renting a specific Compass site.

## Guardrails (non-negotiable)

- Do not fabricate numbers, rankings, or reviews. Real data only; label anything unverified.
- Attribute all contractor credentials to the contractor, never to Compass Camper LLC.
- Do not merge. Do not deploy. No Wrangler. No Cloudflare Direct Upload.
- No em dashes (U+2014) anywhere in the documents.

## Read first (live, at run time)

1. TENANT_ACTIVATION.md from the compass-standards repo (compassleadgroup/Compass-Portfolio-Standards, default branch), via local clone or the available GitHub tools.
2. reference/Rank_and_Rent_Closing_Playbook.md from the same repo.

## Ask the operator for

- Which site this package is for.
- Current ranking or traffic data, if available (GSC export, tracker row, or plain numbers).

## Produce four assets

1. Proof pack. Current organic positions and any traffic evidence for the site, presented plainly. Use only real data the operator provided or that you pulled from a verifiable source. Label anything unverified as unverified. No screenshots of fabricated dashboards, no invented trends.

2. ROI calculator. Estimated lead value for the niche, expected monthly lead volume, and suggested rent. Anchor rent to local CPC times conversion rate times 30 to 50 percent. Show the math line by line so the contractor can check every step. State every assumption.

3. Term sheet. Start with flat monthly rent for a new site with no data. Include pay-per-call and pay-per-lead as options, and note that pay-per-lead carries a higher exit premium and lower onboarding friction. Include term length, exclusivity, and lead cap as open items for negotiation.

4. Prospect list. Licensed local contractors in the niche and metro to approach, with business name, license status if findable, and contact channel. Verify licenses against the state licensing board where possible; label unverified entries.

## Output

Clean, separate documents the operator can send as-is. Compliance rules apply inside them: credentials attributed to the contractor, no fabricated data, no em dashes. Do not deploy anything.
